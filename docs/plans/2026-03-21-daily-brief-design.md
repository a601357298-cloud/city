# 每日早报脚本设计（2026-03-21）

## 目标

做一个本地 Python 脚本，用户只要触发一次，就能自动汇总“今日早报”。输出风格固定为：先给简报，再给重点解读。新闻范围为“国内 + 国际混合”，重点覆盖 AI、经济、国际三类，并补一段对审计/当前工作的启发。

这次设计的重点不是追求抓得最全，而是追求 **稳定产出**。原因很现实：36氪、虎嗅、财联社这类站点经常有连接关闭、风控、动态渲染、DNS 或反爬限制。若把整个流程押在单一网页抓取上，早报就会时灵时不灵。因此脚本要把“抓源”和“成稿”分开，前者允许部分失败，后者必须尽量有结果。

## 方案选择

推荐方案：**多源采集 + 分层兜底 + 本地规则摘要**。

对每个来源，优先走最稳定的接口形态：
- RSS/Atom/公开 JSON 优先
- 其次才是简单 HTML 抽取
- 再不行就走搜索型兜底（保留标题、链接、来源）

相比“只写 requests+BeautifulSoup 爬页面”，这个方案代码稍多，但抗波动能力强。相比“直接接大模型联网搜”，它更可控，也便于后续接 cron 自动化。

## 输出设计

默认输出一份聊天可直接发送的 Markdown 文本，不做长期归档。运行过程中允许写临时文件，但不要求保留日报副本。

成品结构：
1. 今日早报标题 + 生成时间
2. 简报区：AI / 经济 / 国际，每类 3–5 条短摘要
3. 重点解读区：从全部候选里挑 3 条，解释意义
4. 审计联想区：把新闻转成和用户工作相关的启发
5. 数据来源说明：列出成功采集的来源，避免“像编的”

## 架构

脚本建议命名为 `scripts/daily_brief.py`，内部分四层：

- `SourceAdapter`：每个新闻源一个适配器，负责抓取并返回统一条目结构
- `Normalizer`：统一字段（标题、摘要、链接、来源、时间、分类候选）
- `Selector`：去重、打分、按类别挑选条目
- `Renderer`：生成最终 Markdown 文本

统一条目结构至少包含：
- title
- summary
- url
- source
- published_at
- category（AI / economy / world / unknown）
- score

## 新闻源策略

默认主源：
- 国内：36氪、虎嗅、财联社、界面/澎湃（按可用性接入）
- 国际：Reuters、WSJ、FT、Bloomberg（不要求全都成功，按可访问性降级）

策略上允许“名义主源 + 实际降级”。例如：
- 36氪/虎嗅如果页面抓取失败，脚本仍记录“尝试失败”并继续其他源
- Reuters/FT 若正文难取，可先取 headline feed/search 页标题
- 任一类若主源不足，就从其他泛新闻源补足

## 质量与失败处理

必须避免静默失败。

脚本输出中应明确：
- 哪些源成功
- 哪些源失败
- 最终每类用了几条

如果当天抓取很差，脚本仍应输出一版“瘦身早报”，并附一句说明“部分源不可达，已使用备用源补齐”。这样用户至少知道不是系统装死。

## 后续扩展

- 发送到 Feishu
- 引入更强的摘要模型
- 缓存与去重数据库
- 用户自定义关键词偏好
