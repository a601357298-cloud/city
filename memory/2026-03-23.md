# 2026-03-23

- 深夜排查飞书连接异常，最终确认 `open.feishu.cn` 本身没有问题；`curl`、`openssl s_client`、Node HTTPS 都可正常握手。
- 根因锁定为 **Python 证书链环境异常**：Python 3.11 默认 OpenSSL CA 路径指向的 `cert.pem` / `certs` 缺失，导致访问 `open.feishu.cn`、`github.com`、`raw.githubusercontent.com`、`pypi.org` 等多个 HTTPS 站点统一报 `CERTIFICATE_VERIFY_FAILED`。
- 关键诊断经验：若 Python 对多个 HTTPS 站点同时证书失败，而 curl / openssl / Node 正常，应优先怀疑 Python 默认 CA 路径，而不是目标站点。
- 修复验证标准：当 Python 对 `https://github.com` 返回 `200`，对 `https://open.feishu.cn/open-apis/bot/v3/info` 返回 `HTTP 400` 时，说明 TLS 已恢复正常；这里的 400 是业务请求参数问题，不是证书问题。
- 结论已同步写入 MEMORY.md，后续再遇到同类飞书/TLS 问题，优先先查 Python `ssl.get_default_verify_paths()` 与 certifi CA 包路径。
